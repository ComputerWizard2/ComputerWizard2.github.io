关于maven工程部署问题

发表于 2018-07-01 | |
字数统计: | 阅读时长 ≈
  • 工程主要使用的MAven来管理jar包工具
  • taotao-parent：主要是引入工程的所有jar包以及配置编译环境，和服务器的版本：以下是我在工程中积累的经常使用的jar包
  • TaoTao-common都是一个平级的而不是一个工程中的模块，所以采用的是maven-project而不是maven-module.并且该要继承parent这个工程，主要是进行依赖的传递性，将父级的jar包引入到common中，而common中的jar包主要是在项目中所有的可以使用jar。
  • taotao-manager ：他是一个pom的聚合项目，下面分为几个模块：
    • taotao-manager-dao:
      • dao层主要mybatis插件，应该配置的数据源datesource，以及分页插件，数据库的驱动她本身设计不到web页面层所以配置成jar，基本满足需求。她本事依赖与pojo的实体类
      • taotao-manager-interface ：主要是实现服务调用的一些接口，所以同样设计不到web层，所以使用的是jar
    • taotao-manager-service:主要是service层主要依赖dao层和pojo的而它还需要接受来自页面的请求，所以将它设置为一个war包来接受访问：
      （这里强调一下，我们在一个项目特别大的工程里的，我们就需要利用maven来配置一个tomcat依赖：主要是简化了tomcat的配置）配置方式为：
            
- taotao-manager-web:为整个web工程的显示层，所以要建立以一个war包来实现用户的请求访问，这里主要包含spring的jar包的依赖，主要是springmvc还是依赖Spring的，前端的jsp的jar以及jstl等
  • 最后的运行： 运行一个maven依赖的tomcat无论是pom的聚合工程或者是一个war包的web工程都是一样的，配置好tomcat的依赖，以及需要配置的工程名和对应的端口号，启动的时候：右击工程名选择对应的run —>manven build .. 在输入框内： clean tomcat7: run ,但是一般这样运行都会报错，我们需要确定每一个依赖的jar包都在仓库里都能找的到，未找到的我们需要 build install 进行写入到仓库，这样这个工程基本就可以运行了
