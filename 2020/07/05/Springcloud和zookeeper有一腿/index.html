Springcloud和zookeeper有一腿

发表于 2020-07-05 | |
字数统计: | 阅读时长 ≈
  • 基本就是那个套路
  • 分为客户端和服务端，还有一个注册器，只不过注册器是以一个外部运行的应用实现的，所以工程里没有服务注册器的实现
  • 服务端：
    • pom文件主要多引入zookeeper客户端的jar包，注意zookeeper的springcloud jar引入了一个低版本的jar 需要排除掉。
  • yml文件没有什么就是配置了端口号和远程访问zookeeper的地址
  • application主要实现服务的注册服务的注解
  • controller没有什么业务逻辑
  • 客户端：
    • pom文件和服务段差不多
  • yml文件和服务端区别不大
  • aoolication
  • 调用服务端的controller
  • 配置类配置了restTemplete实现rest风格

总结一下：今天学习了Springcloud和zookeeper作为我们的注册器，配置起来相对来说比较简单，zookeeper对service的态度没有那么好，可以说是0容忍，只要你他娘的挂掉，老子就把你踢走，你恢复了心跳，我就给你加回来，不像eureka那么的宽容，你心跳停止了，我还等你几秒，看你还能不能活过来。大致的也就了解这么多，我的工程就是端口8004和Springcloud-consummer-order的两个工程项目。GitHub地址：https://github.com/ComputerWizard2/springcloud-learn
