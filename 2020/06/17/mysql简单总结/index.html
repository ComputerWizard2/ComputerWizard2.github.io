  • 事物 ACID的四大特性

    • 持久性 事物只要不提交，那么就可以回退，如果提交了那么就不可逆了。

    • 隔离性：不会被其他事物干扰，单独开启一个事物

    • 原子性：要么都成功，要么都失败

    • 一致性：事物前后的数据的完整性要保持一致

  • 隔离出现的问题：

    • 脏读
    • 不可重复读
    • 幻读

  • 事物经常的用到的语句

    • 关闭自动提交 set autocommit=0

    • 关闭自动提交 set autocommit=1
    • 设置一个保存点 savepoint 保存点名
    • rollback to savepoint 保存点
    • release savepoint 保存点名
  • 模拟转账事物
    • 关闭自提交 set autocommit = 0；
    • 开启一个事物 start transaction
    • 事务中的操作： update account set money =money-500 where name=‘A’
    • update account set money =money+500 where name=‘B’
    • 提交事物 commit

    • 恢复自动提交 set autocommit =1
